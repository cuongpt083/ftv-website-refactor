# CODEX IMPLEMENTATION BRIEF
Project: FINTECHVIET Website  
Objective: Generate HTML templates for UI review before production development.

---

# 1. Project Goal

You are an AI coding agent responsible for generating static HTML templates for a fintech enterprise website.

These templates will be used for:

- UI review
- UX validation
- responsive testing
- design approval

The templates will later be implemented using:

Next.js + Strapi (Headless CMS)

However, **this phase ONLY generates static HTML templates.**

Do NOT implement backend, CMS, or Next.js code.

# 4. HTML Requirements

Use semantic HTML5.

Required elements:

header  
nav  
main  
section  
article  
footer  

Accessibility rules:

All images must have alt attributes.

Buttons must have accessible labels.

Forms must include labels.

---

# 5. CSS Requirements

CSS must follow a mobile-first approach.

Structure:

tokens.css
components.css
layout.css

styles.css inside each template may import shared styles.

Rules:

Do NOT hardcode colors.

Use tokens defined in design-tokens.md.

Example:

--color-primary  
--spacing-md  
--radius-lg

---

# 6. Responsive Behavior

Follow breakpoints defined in design-tokens.md.

Mobile-first layout:

Default = mobile

Tablet = min-width 640px

Desktop = min-width 1024px

Rules:

Cards stack on mobile.

Navigation collapses into mobile drawer.

Logo strips scroll horizontally.

Case study cards may use horizontal scroll on mobile.
